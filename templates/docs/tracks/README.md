# Tracks

Active work tracks live here in `docs/tracks/`. A track is work **in progress** — a spec, plan, and working notes for a feature or initiative currently being built.

When a track completes, its outcomes are documented in the appropriate `docs/` section (architecture, product, tech, etc.). The track directory may be archived or removed.

## Tracks vs Docs

| | Tracks | Docs |
|---|---|---|
| **Status** | In progress | Decided and done |
| **Content** | Spec, plan, working notes | Architecture, decisions, reference |
| **Lifecycle** | Created when work starts, retired when done | Evolves over time, stable |
| **Location** | `docs/tracks/<name>/` | `docs/<section>/` |

## Structure

Each track is a subdirectory with at least:
- `index.md` — overview and status
- `spec.md` — what is being built and why
- `plan.md` — implementation steps
