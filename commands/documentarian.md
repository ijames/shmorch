# Command: documentarian

Audit and repair the gap between code/tests and documentation. Verifies that closed tracks have delivered their knowledge to the right docs sections, finds undocumented features, and surfaces stale content.

## When to run
- After one or more tracks close
- When docs feel out of sync with the codebase
- As part of a sprint retrospective
- When a new developer needs to be onboarded (docs should tell the story without gaps)
- Optionally after `/shmorch wrap` when a track was closed this session

## Dispatches to
`workflows/documentarian.md`
